OpenGIS® interfaces for Java. Interfaces were created from UML diagrams found in the following specifications:
  • Metadata (Topic 11)
  • Catalog Services
  • Spatial Referencing by Coordinates (Topic 2) completed by Coordinate Transformation Services
  • Feature Geometry (Topic 1)
  • Grid Coverages Implementation
  • GO-1 Application Objects Report

Interfaces are created using a fixed set of UML to Java mapping. Substantial modifications are declared in the list of changes from UML diagrams.

Status

  • Spatial Referencing by Coordinates
    Mostly completed and should be stable. Minor additions expected, especially something equivalent to the legacy WGS84ConversionInfo structure from OGC 01-009.

  • Feature Geometry
    Geometries partially done. New classes and methods to be added, especially related to topology. The package names may be revisited.

  • Grid coverage
    Mostly completed and stable until the next OGC specification, excepted GridCoverageExchange, Reader and Writer which are under revision.

  • Metadata
    Placeholders for resolving dependencies in Spatial Referencing by Coordinates. Will be completed later.

  • User interfaces
    First submission. Those packages are not derivated from an OGC abstract specification, but are themselves a proposal for a specification. They may be revisited as more experience will be gained.

Requirements

  • J2SE 1.3 or later.
  • A JSR-108 units implementation.
